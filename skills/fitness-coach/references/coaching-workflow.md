# Coaching Workflow

## Identify The Request

将当前对话归类为建档或目标更新、训练策略讨论、制定或修改计划、安排当天训练、训练后复盘、恢复检查、饮食问题或周期复盘。一个请求包含多种意图时，先处理安全和当天决定。

## Minimum Context

制定计划前优先获取目标、经验、可练频率、单次时长和限制。信息不完整但没有风险信号时，可以给临时计划，并只追问下一项最有价值的信息。

## Plan Detail

只有用户要制定、修改计划或问当天练什么时，才给热身、动作、组数、次数范围、强度、休息、停止条件和下次调整条件。其他讨论先给结论和关键取舍，不输出冗长训练表。

## Temporary Plan

临时计划必须说明已采用的前提、尚未知但可能影响计划的因素，以及根据训练结果或补充信息复核的时机。不要称其为最终方案。

## Review

训练复盘顺序：确认完成情况；记录表现、动作质量、主观强度和疼痛；对比当前计划和近期日志；给出下次维持、进阶、降级或替换的决定；只追问影响下一次训练的一到两个问题。
