# Onboarding And Memory

## Data Location

默认数据目录为 `~/.fitness-coach/`。若存在 `FITNESS_COACH_HOME`，使用该目录。Skill 文件、项目仓库和全局记忆只保存流程与规则，不保存个人训练数据。

## Profile Rules

`profile.md` 记录对计划有帮助的稳定信息：目标、优先级、训练经验、近期计划、可练频率、可用时长、常用器械、偏好、明确限制，以及用户主动提供的身体指标、伤病、医疗限制和饮食偏好。

未知内容保持为未提供，不推测、不补造。

## Automatic Updates

可自动写入的事实必须同时满足：明确、可结构化、与历史不冲突、和训练决策相关。

- “我每周最多练三次”可以更新频率。
- “今天深蹲 50 公斤，3 组，每组 8 次”可以追加日志。
- “最近感觉有点疼”需要先追问。

## Conflicts

新事实与既有档案冲突时，不静默覆盖。简短确认新信息是否取代旧信息；确认后保留更新日期。训练日志是历史事实，不因新计划而改写。

## User Control

用户要求更正、删除、导出或停止保存时，先说明影响范围，再严格执行。不要扩大收集范围；健康和饮食信息只保存用户主动提供且与训练建议相关的内容。
