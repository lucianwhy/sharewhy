Reading-Oriented Layout

把内容排成技术说明页，而不是后台卡片墙

这个 preview 展示的不是业务内容，而是 `html-use` 的默认气质：暖白底、衬线标题、等宽路径、安静配色、左侧可收缩导航。适合总结、方案、复盘、代码理解页。

Walkthrough Blocks

1
summary / 先讲结论

开头先给一段总结，告诉读者这页到底想说明什么，不要让人自己猜重点。

2
walkthrough / 再展开链路

如果是架构、流程、复盘，优先拆步骤写，而不是一大段散文。

3
key files / 最后给定位锚点

如果内容和代码或文档有关，用 `key files`、`gotchas`、`next step` 让读者快速继续行动。

Key Files

  • html/feature-scope-review.html 用户最终打开的 HTML 产物。
  • skills/html-use/SKILL.md 约束输出目录、导航结构、默认风格和验证方式。
